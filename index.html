Warstwy TCP/IP

Model TCP/IP składa się z 4 warstw. Każda warstwa realizuje określone zadania i współpracuje z pozostałymi podczas przesyłania danych przez sieć.

  1. Application Layer Warstwa aplikacji - DNS, HTTP, HTTPS, SMTP, POP3, IMAP, FTP, SFTP, TFTP, REST
  2. Transport Layer Warstwa transportowa - TCP i UDP, porty, segmenty, kontrola błędów
  3. Internet Layer Warstwa internetu - IP, IPv4, IPv6, NAT, ICMP, OSPF, EIGRP, BGP
  4. Network Access Layer Warstwa dostępu do sieci - ARP, Ethernet, VLAN, MAC, Frame, Switch, NIC

TCP kontra UDP

TCP jest połączeniowy, niezawodny, potwierdza odbiór danych, kontroluje błędy i zachowuje kolejność pakietów.

UDP jest bezpołączeniowy, bardzo szybki, ma małe opóźnienia i nie gwarantuje dostarczenia pakietów.

Najkrótsze skojarzenia

ARP: IPv4 na MAC.

DNS: domena na IP.

Routing: wybór trasy pakietu.

VLAN: logiczny podział jednej sieci fizycznej.

Frame: ramka w warstwie dostępu do sieci.
